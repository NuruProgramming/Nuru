# Security Policy

## Reporting a vulnerability

If you believe you have found a security vulnerability in Nuru, please report it so we can address it.

**Preferred:** Open a [GitHub Security Advisory](https://github.com/NuruProgramming/Nuru/security/advisories/new) (allows private discussion and coordinated disclosure).

**Alternative:** Open a [GitHub Issue](https://github.com/NuruProgramming/Nuru/issues) and label or title it so maintainers can treat it with appropriate confidentiality. For highly sensitive issues you may contact the maintainers via the repository owner profile.

**Please include:**

- A description of the vulnerability and how it might be exploited
- Steps to reproduce (if applicable)
- Your Nuru version and environment
- Any suggested fix or mitigation (optional)

**What to expect:**

- We will acknowledge your report and aim to respond within a reasonable time.
- We will work with you to understand and address the issue.
- We may publish a security advisory or release a patch; we will credit you unless you prefer to remain anonymous.

Please do not disclose the vulnerability publicly before we have had a chance to address it.

## Supported versions

We release security updates for the current and recent release line. Upgrade to the latest version when possible.

## Security-related design

For design decisions that affect security (e.g. sandboxing, module restrictions), see [docs/POLICY.md](docs/POLICY.md) and [docs/upgrade_nuru_road_to_bigLeague/README.md](docs/upgrade_nuru_road_to_bigLeague/README.md) where relevant roadmap items are tracked.
